Architecture

How it works end-to-end

The script has three distinct layers: geometry reading, solid construction, and Civil 3D object writing. Dynamo bridges all three without leaving the Civil 3D session.

01
Read Source Geometry
Extract vertices, station distances, and tangent vectors from the selected 3D polyline or feature line via the Civil 3D COM API.
02
Apply Dash Pattern
Walk the line by cumulative distance, splitting it into ON (dash) and OFF (gap) intervals per the SN 640850A lookup table for the chosen type.
03
Sweep & Union
For each dash segment: create a rectangular profile, sweep it along the sub-curve, then Boolean-union all swept solids into one object.
04
Write & Link
Write the solid to the DWG as a Civil 3D feature, store the source object Handle as XData, enabling re-run regeneration without manual selection.

Workflow

User workflow inside Civil 3D

1
Draw the 3D Polyline or Feature Line
Assign Z-elevations to all vertices. Feature lines are preferred — they carry chainage data and can follow a corridor baseline exactly.
2
Open the Dynamo Player
Launch the graph from the Civil 3D Manage tab → Dynamo Player. No need to open the full Dynamo editor each time.
3
Select the line, choose type & width
A Select Object node prompts you to click the line. A Dropdown node lists M1–M10. An optional Number Slider overrides the default width.
4
Run
Click Run. The Python Script node executes, creates the solid, and writes it to a dedicated layer (e.g. C-ROAD-MARK).
5
Re-run after line edits
Edit the source polyline grip points. Re-run the graph — the script finds the old solid via its XData handle, deletes it, and regenerates. No manual cleanup.

Full Python script — paste into the Python Script node

road_marking.py — Python Script Node 
# ============================================================
# Road Marking Generator — Dynamo Python Script Node
# Standard: SN 640850A (adapt constants for other standards)
# Author:   Your Name / Your Firm
# Requires: Civil 3D 2022+ with Dynamo 2.x, IronPython 2.7
# ============================================================

import clr
import sys
import math

clr.AddReference('AcMgd')
clr.AddReference('AcDbMgd')
clr.AddReference('AecBaseMgd')
clr.AddReference('AeccDbMgd')   # Civil 3D objects

from Autodesk.AutoCAD.ApplicationServices import Application
from Autodesk.AutoCAD.DatabaseServices import (
    Transaction, OpenMode, Polyline3d, Entity,
    BlockTableRecord, ResultBuffer, TypedValue,
    DxfCode, SolidHistoryMode
)
from Autodesk.AutoCAD.Geometry import (
    Point3d, Vector3d, Plane, Matrix3d
)
from Autodesk.Civil.DatabaseServices import FeatureLine

# ── DYNAMO INPUTS ─────────────────────────────────────────────
# IN[0]: selected Civil 3D object (from Select Object node)
# IN[1]: marking type string  e.g. "M1"
# IN[2]: width override float (0 = use standard default)

source_obj     = IN[0]   # Dynamo-unwrapped element
mark_type      = IN[1]   # e.g. "M1"
width_override = IN[2]   # 0.0 means use default

# ── MARKING STANDARD LOOKUP TABLE (SN 640850A) ───────────────
# Keys:  type name
# Values: (dash_m, gap_m, default_width_m, thickness_m)
#         dash_m = 0 → continuous solid line

STANDARDS = {
    "M1":  (3.0,  9.0,  0.12, 0.006),
    "M2":  (0,    0,    0.15, 0.006),  # solid
    "M3":  (0,    0,    0.15, 0.006),  # solid (double handled externally)
    "M4":  (1.0,  1.0,  0.30, 0.006),
    "M5":  (1.0,  3.0,  0.12, 0.006),
    "M6":  (0,    0,    0.12, 0.006),
    "M7":  (0,    0,    0.25, 0.006),
    "M8":  (1.5,  0.5,  0.10, 0.006),
    "M9":  (0,    0,    0.50, 0.006),
    "M10": (0,    0,    0.50, 0.006),
}

MARK_LAYER  = "C-ROAD-MARK"
XDATA_APP   = "ROAD_MARK_SRC"   # registered app name for XData

# ── HELPERS ───────────────────────────────────────────────────

def get_polyline_points(tr, obj_id):
    """Return list of Point3d from a 3dPolyline or FeatureLine."""
    obj = tr.GetObject(obj_id, OpenMode.ForRead)
    pts = []
    if isinstance(obj, FeatureLine):
        for i in range(obj.PointsCount):
            pts.append(obj.GetPoint(i))
    elif isinstance(obj, Polyline3d):
        for vtx_id in obj:
            vtx = tr.GetObject(vtx_id, OpenMode.ForRead)
            pts.append(vtx.Position)
    return pts


def segment_lengths(pts):
    """Return list of (cumulative_start, length, pt_a, pt_b)."""
    segs = []
    cum = 0.0
    for i in range(len(pts) - 1):
        a, b = pts[i], pts[i+1]
        d = a.DistanceTo(b)
        segs.append((cum, d, a, b))
        cum += d
    return segs, cum


def lerp_point(a, b, t):
    """Linearly interpolate between two Point3d values."""
    return Point3d(
        a.X + (b.X - a.X) * t,
        a.Y + (b.Y - a.Y) * t,
        a.Z + (b.Z - a.Z) * t,
    )


def point_at_distance(segs, dist):
    """Return Point3d at a given cumulative distance along segments."""
    for (cum_s, seg_d, a, b) in segs:
        if dist <= cum_s + seg_d + 1e-6:
            t = (dist - cum_s) / seg_d if seg_d > 0 else 0
            return lerp_point(a, b, max(0, min(1, t)))
    return segs[-1][3]   # end of line


def build_dash_intervals(total_len, dash, gap):
    """
    Return list of (start_dist, end_dist) for each ON segment.
    If dash == 0: single interval spanning full length (solid).
    """
    intervals = []
    if dash == 0:
        intervals.append((0.0, total_len))
        return intervals
    cycle = dash + gap
    pos = 0.0
    while pos < total_len:
        end = min(pos + dash, total_len)
        if end - pos > 0.01:   # skip slivers
            intervals.append((pos, end))
        pos += cycle
    return intervals


def create_box_solid(db, tr, msp, pt_a, pt_b, width, thickness):
    """
    Create a 3D Solid box representing one dash segment.
    The box is oriented along the vector pt_a → pt_b,
    width is perpendicular in the XY plane, thickness is in Z.
    Returns the ObjectId of the created solid.
    """
    vec = Vector3d(pt_b.X - pt_a.X, pt_b.Y - pt_a.Y, pt_b.Z - pt_a.Z)
    length = vec.Length
    if length < 0.001:
        return None
    vec = vec.GetNormal()

    # Local coordinate system
    up    = Vector3d.ZAxis
    right = vec.CrossProduct(up).GetNormal()   # width direction
    if right.Length < 0.5:                       # vec nearly vertical
        right = Vector3d.XAxis

    hw = width / 2.0

    # Eight corners of the box
    corners = [
        pt_a + right * hw,
        pt_a - right * hw,
        pt_b - right * hw,
        pt_b + right * hw,
    ]

    # Build solid via Acad3DSolid (box method is simpler via COM;
    # here we use the managed API's AddBox then transform)
    solid = Autodesk.AutoCAD.DatabaseServices.Solid3d()
    solid.CreateBox(length, width, thickness)

    # Build transform: origin at midpoint, X along vec
    mid = Point3d(
        (pt_a.X + pt_b.X) / 2,
        (pt_a.Y + pt_b.Y) / 2,
        (pt_a.Z + pt_b.Z) / 2 + thickness / 2,
    )
    mat = Matrix3d.AlignCoordinateSystem(
        Point3d.Origin, Vector3d.XAxis, Vector3d.YAxis, Vector3d.ZAxis,
        mid, vec, right, Vector3d.ZAxis
    )
    solid.TransformBy(mat)

    solid.Layer = MARK_LAYER
    solid.ColorIndex = 7   # White

    msp.AppendEntity(solid)
    tr.AddNewlyCreatedDBObject(solid, True)
    return solid.ObjectId


def delete_old_solid(db, tr, msp, src_handle_str):
    """
    Find and erase a previously generated solid that stores
    src_handle_str in its XData (app: ROAD_MARK_SRC).
    """
    for entity_id in msp:
        ent = tr.GetObject(entity_id, OpenMode.ForRead)
        xdata = ent.GetXDataForApplication(XDATA_APP)
        if xdata is None:
            continue
        for tv in xdata:
            if tv.TypeCode == 1000 and tv.Value == src_handle_str:
                ent.UpgradeOpen()
                ent.Erase()
                break


# ── MAIN ──────────────────────────────────────────────────────

def run():
    doc = Application.DocumentManager.MdiActiveDocument
    db  = doc.Database
    ed  = doc.Editor

    if mark_type not in STANDARDS:
        return f"ERROR: Unknown marking type '{mark_type}'"

    dash, gap, default_w, thickness = STANDARDS[mark_type]
    width = float(width_override) if width_override else default_w

    with doc.LockDocument():
        with db.TransactionManager.StartTransaction() as tr:
            bt  = tr.GetObject(db.BlockTableId, OpenMode.ForRead)
            msp = tr.GetObject(bt[BlockTableRecord.ModelSpace], OpenMode.ForWrite)

            # Unwrap Dynamo element to get ObjectId
            try:
                obj_id = source_obj.InternalObjectId
            except:
                return "ERROR: Could not unwrap source object."

            src_handle = tr.GetObject(obj_id, OpenMode.ForRead).Handle.ToString()

            # Remove previous solid generated from this source line
            delete_old_solid(db, tr, msp, src_handle)

            # Ensure layer exists
            lt = tr.GetObject(db.LayerTableId, OpenMode.ForWrite)
            if not lt.Has(MARK_LAYER):
                from Autodesk.AutoCAD.DatabaseServices import LayerTableRecord
                ltr = LayerTableRecord()
                ltr.Name = MARK_LAYER
                lt.Add(ltr)
                tr.AddNewlyCreatedDBObject(ltr, True)

            # Ensure XData app is registered
            rat = tr.GetObject(db.RegAppTableId, OpenMode.ForWrite)
            if not rat.Has(XDATA_APP):
                from Autodesk.AutoCAD.DatabaseServices import RegAppTableRecord
                ra = RegAppTableRecord()
                ra.Name = XDATA_APP
                rat.Add(ra)
                tr.AddNewlyCreatedDBObject(ra, True)

            # Get points along the source line
            pts = get_polyline_points(tr, obj_id)
            if len(pts) < 2:
                return "ERROR: Source line has fewer than 2 vertices."

            segs, total_len = segment_lengths(pts)
            intervals = build_dash_intervals(total_len, dash, gap)

            if not intervals:
                return "ERROR: No dash intervals generated — line too short?"

            solid_ids = []
            for (s, e) in intervals:
                pa = point_at_distance(segs, s)
                pb = point_at_distance(segs, e)
                sid = create_box_solid(db, tr, msp, pa, pb, width, thickness)
                if sid:
                    solid_ids.append(sid)

            if not solid_ids:
                return "ERROR: No solids created."

            # Boolean-union all solids into the first one
            base = tr.GetObject(solid_ids[0], OpenMode.ForWrite)
            for i in range(1, len(solid_ids)):
                tool = tr.GetObject(solid_ids[i], OpenMode.ForWrite)
                base.BooleanOperation(
                    Autodesk.AutoCAD.DatabaseServices.BooleanOperationType.BoolUnite,
                    tool
                )
                tool.Erase()

            # Stamp XData on the unified solid so we can find it on re-run
            rb = ResultBuffer(
                TypedValue(1001, XDATA_APP),       # app name
                TypedValue(1000, src_handle),       # source handle string
                TypedValue(1000, mark_type),        # type for reference
            )
            base.XData = rb

            tr.Commit()

    return (
        f"OK: {mark_type} marking created. "
        f"{len(intervals)} dashes unified into 1 solid. "
        f"Total length: {total_len:.2f} m, Width: {width:.3f} m."
    )

OUT = run()
Deep Dive

Key techniques explained

Boolean Union — the "single solid" trick

Each dash is created as an independent Solid3d. After all dashes exist in the database, a loop calls BooleanOperation(BoolUnite, tool) on the first solid against each subsequent one, then erases the consumed tool. The result is a single Solid3d with internal history stored as BREP shells.

Dynamic link via XData

XData is AutoCAD's per-object key-value store. The script writes the source line's Handle (a permanent string ID, unlike ObjectIds which can change) into the solid's XData under a registered application name. On re-run, delete_old_solid() scans model space, finds the solid whose XData matches the current source handle, erases it, then regenerates fresh — giving the illusion of a live parametric link.

Coordinate transform for orientation

CreateBox() always produces a world-axis-aligned box. Matrix3d.AlignCoordinateSystem() rotates and translates it so its local X-axis aligns with the dash direction vector, placing it correctly in 3D space even on curved or graded alignments.

For curved alignments: Increase the sampling resolution in get_polyline_points() by subdividing each segment. Feature lines from a corridor already carry dense vertex arrays — this is why they're preferred over simple polylines for curved roads.

Improving for real production use

Curved segments
Sample each polyline segment at every 0.5 m rather than just at vertices. This approximates curves and avoids flat-chord artifacts on tight radii.
Layer + color by type
Assign different layers and True Colors per marking type to support automated plan production and quantity takeoffs in Civil 3D.
Double lines (M3)
Run the script twice with ±offset to the source line, or add an offset parameter that creates two parallel solids and unions them together.
Dynamo Player params
Expose mark_type and width via Dynamo Player's input panel so users never open the full editor — just click Run in the Player window.
Geometry

Alignment and horizontal geometry

The alignment is the master horizontal control: tangents, circular curves, and spirals (clothoids) where your design standard requires transition length. Stations, station equations, and design speed feed everything downstream (profile, superelevation, corridors, labels).

H1
Civil 3D objects
Alignment entities, design criteria sets (AASHTO-style or custom), fixed/floating/free curves and spirals, superelevation critical stations.
H2
Automation levers
Import PI/curve tables from CSV or LandXML; Dynamo to place alignments from polylines; .NET for batch criteria application and QA checks on radius and spiral lengths.
H3
Outputs
Labeled plans, legal stationing, offset baselines for lanes and services, geometry reports for checks against standard templates.
H4
BOQ link
Indirect: chainage and horizontal extents drive length-based pay items (pavement, barriers) once tied to corridor or marking runs.

Typical workflow

1
Define PIs and design criteria
Create the alignment from COGO points, polyline fit, or imported geometry; attach a criteria set for min radius, transition types, and design speed.
2
Tune curves and spirals
Resolve curve groups; add spirals where required; use station equations at structure or jurisdiction boundaries.
3
Publish for downstream models
Data shortcuts or xref templates; sample feature lines at offsets for markings, kerbs, and drainage laterals.
Geometry

Vertical profile and vertical curves

The profile ties elevation to station along the alignment using tangents and parabolic vertical curves at PVIs. Grades control drainage, sight distance, and earthwork balance; profiles can represent existing ground (EG) or finished grade (FG) for corridors.

V1
Civil 3D objects
Surface profiles, layout profiles, profile views, PVI-based layout, vertical curve K-values or length, band sets for slope and elevation.
V2
Automation levers
Profile from file or Excel grades; Dynamo to place PVIs at regular chainage; minimum grade rules scripted for stormwater checks.
V3
Outputs
Profile sheets, clash checks against utilities, FG profiles consumed by corridor assemblies for pavement structure.
V4
BOQ link
Indirect: vertical design sets finished levels that drive corridor volumes and material thickness when tied to structure depth.

Typical workflow

1
Attach EG surface
Create a surface profile on the alignment; use it as context for cut and fill in the profile view.
2
Layout FG PVIs and vertical curves
Meet max/min grades and K-value standards; coordinate superelevation crossfall with edge-of-travel-way elevations where required.
3
Bind to corridor
Use the finished profile as a corridor baseline for pavement and structure; regenerate when horizontal or vertical edits land.

Commercial

Quantity takeoff and BOQ generation

Quantity takeoff (QTO) aggregates lengths, areas, and counts from alignments, corridors, surfaces, and AutoCAD solids. Map Civil 3D codes and layers to pay items, then export to CSV or Excel for the Bill of Quantities (BOQ) used in tendering and progress claims.

Q1
Civil 3D objects
QTO Manager, corridor quantity takeoff, material lists, pipe runs, AutoCAD solids by layer for custom items (markings).
Q2
Automation levers
Pay item formula tables; export to CSV; Dynamo to sum solid volumes or lengths by type from XData; Power Query in Excel to shape BOQ columns.
Q3
Outputs
BOQ workbooks with sections for earthworks, pavement, drainage, markings, signs; revision logs keyed to drawing revision.
Q4
BOQ link
Direct: this stage is the BOQ source of truth when pay mapping and units are validated against the contract schedule.

Typical workflow

1
Define pay item mapping
Link corridor codes, pipe sizes, and solid layers to contract item numbers and measurement units (m, m², m³, each).
2
Run QTO and corridor reports
Refresh after corridor rebuild; isolate by region or phase for staged tenders.
3
Export and format BOQ
Push totals into Excel BOQ template; archive CSV with drawing issue for audit.
