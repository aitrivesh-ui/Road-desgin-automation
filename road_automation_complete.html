Summary

At a glance

This repository automates repetitive Civil 3D road design steps using CSV tables, a single config/project.json, and Dynamo Python Script nodes running IronPython 2 inside Civil 3D. A separate marking workflow (documented in the markings hub) builds pavement marking solids from 3D source geometry.

  1. Prepare once: a template DWG/DWT with layers, Civil styles, assemblies, sign blocks, and surfaces named consistently with your config.
  2. Prepare per job: copy CSV templates into civil3d_automation/csv/, fill data; ensure config/project.json exists (copy from project.example.json, or run Windows install.bat / civil3d_automation\install_tools.bat once to bootstrap project.json if missing), then align every name and path.
  3. Validate: run preflight (GUI or --validate) before opening Civil 3D.
  4. Execute: open the project drawing, run m8_run_all.py from Dynamo Player with the absolute path to project.json, then run marking graphs per line as needed.
  5. Deliver: review out/ CSVs (volumes, BOQ), QA logs, and the drawing itself before issue.
Expectations

Scope and limits

The automation applies your numbers and naming. It does not choose design standards, interpret the contract, or certify the model. Curve and spiral groups in horizontal geometry are a polyline-style line chain in M1 unless you extend the script; vertical curves follow CSV lengths and K fields where implemented in M2.

Engineering responsibility: treat script output as a draft model and quantity set. Commercial rounding, progress claims, and construction tolerances remain human-controlled.
System

Architecture and data flow

Two parallel tracks work together:

  • M1–M8 package (civil3d_automation/python/): alignment, profile, corridor regions, volumes, signage, sheet helper stub, BOQ rollup, and orchestrator M8.
  • Marking module (Markings hub in this handbook): one script per graph, dashed or solid markings from feature lines or 3D polylines, with XData for safe re-run.

Typical order: M1 horizontal, M2 vertical, M3 corridor regions, M4 volumes CSV, marking (per source line), M5 signage, M6 checklist, M7 BOQ. M8 runs M1–M7 in sequence when you pass no step filter.

Onboarding

Day-zero checklist

  1. Install Civil 3D + Dynamo; confirm IronPython 2 on Python Script nodes.
  2. Build or import the template (layers, styles, label sets, assembly, blocks, surfaces).
  3. Clone or copy the automation folder; ensure config/project.json (copy from project.example.json, or run install.bat / civil3d_automation\install_tools.bat on Windows if project.json is absent).
  4. Copy csv/templates/*.csv into csv/ and fill rows (UTF-8 CSV preferred).
  5. Run install.bat or civil3d_automation\install_tools.bat once if you have not yet installed Python 3 tool dependencies; then run civil3d_automation/tools/run_preflight.bat or CLI validate (see cheat sheet).
  6. Wire Dynamo graphs per README_BUILD_GRAPHS.md; test M1 alone, then M2, then chain.
  7. Record Civil year, Dynamo build, and Python engine in API_NOTES.md after first green run.

In Civil

Dynamo graphs and M8

For each module, create a graph with one Python Script node: paste the entire matching python/m*.py, wire inputs in IN[0] order from the file docstring, attach a Watch to OUT, save under civil3d_automation/dynamo/ if desired, and publish to Dynamo Player.

M8 inputs: IN[0] = absolute path to config/project.json. IN[1] = optional comma-separated step filter (case-insensitive). Empty IN[1] runs m1,m2,m3,m4,m5,m6,m7.

Example IN[1]Effect
(empty)Full chain M1 through M7
m1,m2Alignment and profile only
m4Corridor rebuild and volumes only
m5,m7Signage then BOQ (prerequisites must exist)

Label String nodes in Dynamo with the same phrases as the docstrings so operators know what to paste. See README_BUILD_GRAPHS.md.

Operations

Run modes and QA order

Production (full chain)

  1. Preflight CSV + JSON.
  2. Open project DWG from template.
  3. Run M8 with absolute JSON path.
  4. Run marking graph for each marking source line.
  5. Re-run M7 alone if only BOQ inputs changed (IN[1] = m7).

Debugging (stepwise)

Run M1, confirm alignment in Prospector, then M2, and so on. Use M8 step filters once individual steps are trusted.

Strict data validation

Add --strict to preflight CLI or enable strict in the GUI to enforce: at least two alignment PI rows, numeric stations, strictly increasing profile stations, valid region station ranges, and start_sta < end_sta per region.

Geometry

Pavement markings

The marking workflow is separate from M8: open the Markings hub section for the script, XData app names (ROAD_MARK_SRC), standard dash tables, and recommended placement in the pipeline (typically after corridor model and before or alongside signage depending on your QA).

Re-runs

Idempotency and re-runs

  • M5 erases prior inserts tagged with the same XData app (default ROAD_SIGN_CSV) before placing again.
  • M1 / M2 fail if the alignment or profile name already exists — delete or rename in Civil before re-run.
  • M3 adds regions; duplicate region names may error — adjust CSV or clear regions manually.
  • Marking script removes prior geometry it tagged so you can regenerate from the same source line.
Process

Version control and drawing issues

  • Branch or tag project.json and csv/*.csv with each drawing issue (for example issue/REV-B).
  • Prefer repo-relative paths in JSON as in project.example.json; avoid committing machine-specific absolute paths.
  • Commit last_run_manifest.json only if your QA policy needs it; otherwise ignore per team rules.
  • If you commit .dyn files, record Civil 3D year, Dynamo build, and Python engine in API_NOTES.md — graphs are build-sensitive.

Overview

What this automation is

This workflow targets Autodesk Civil 3D projects where repetitive tasks (alignments from tables, profiles from grades, corridor rebuilds, volumes, signage placement, BOQ rollups, and pavement marking solids) are driven by CSV data and Dynamo Python Script nodes instead of hundreds of manual picks and dialog steps.

Two documentation layers work together:

  • Marking module — described in the Markings hub section: one Python script creates SN-style dashed/solid marking solids from a 3D polyline or feature line, with XData for re-run without duplicate geometry.
  • M1–M8 package — implemented under civil3d_automation/: separate scripts for alignment, profile, corridor regions, volumes, signage, sheet helper, BOQ rollup, and an orchestrator (m8_run_all.py) that reads config/project.json.
Important: Nothing here replaces engineering judgement. Automation applies your standards (CSV numbers, template styles, pay-item map) and still needs you to verify geometry, contract mapping, and issue control.

There is no single MSI that installs Civil 3D automation for you: you install Autodesk Civil 3D and Dynamo separately, then use this repository’s scripts and data. For the optional Python 3 helpers (preflight GUI, unit tests, Excel workbook export), Windows users can double-click install.bat at the repo root or civil3d_automation\install_tools.bat — that runs pip install -r requirements-tools.txt and creates config/project.json from project.example.json only if project.json is missing.

One-time

First-time setup — template drawing

Automation assumes a template DWG or DWT that already contains Civil styles and CAD primitives the scripts reference by name. Full checklist is in civil3d_automation/template/README.md. Summarised here:

  • Layers: e.g. 0, C-ROAD-MARK, C-SGN-FURN (match project.json → names.layers).
  • Alignment & profile styles + label sets: names must match project.json → styles.* (defaults often Standard).
  • Assembly: an assembly whose name matches names.assembly (used by M3).
  • Blocks (M5): block definitions exactly as listed in signage_schedule.csv (e.g. SGN_STOP).
  • Surfaces (M4/M7): TIN (or compatible) surfaces for existing ground and finished model, named e.g. EG and FG per config.
  • Feature lines (optional M3 extension): if you later bind targets from CSV, name them consistently (e.g. EOP_L).

Creating this template is manual Civil 3D work once per firm or project type; afterwards each job mainly edits CSV/JSON and runs Dynamo.

Operations

Steps to follow each design run

Option A — Full chain (M8)

  1. Open your project DWG created from the template.
  2. Update csv\ files and config\project.json (alignment PI, profile PVIs, regions, signage, pay items, output paths).
  3. Open Dynamo Player and run the graph that contains m8_run_all.py with IN[0] = full path to project.json.
  4. Optional IN[1]: comma list of steps, e.g. m1,m2,m4 to run a subset.
  5. For each marking source line, run the marking graph from the Markings hub section (between corridor work and signage/BOQ is typical).
  6. Open out\ CSVs in Excel; perform design QA and issue PDFs/sheets as per your QA procedure.

Option B — Step-by-step (M1 then M2 …)

  1. M1 — PI CSV → alignment (polyline-style fixed lines unless you extend curves).
  2. M2 — PVI CSV → layout profile on that alignment; read grade QA in Watch.
  3. M3 — Region table → corridor baseline regions + rebuild.
  4. M4 — Rebuild corridor; create volume surface EG vs FG; write volume CSV.
  5. Marking — per feature line / 3D polyline (Dynamo graph; script in Markings hub section).
  6. M5 — Signage CSV → blocks (re-run safe: prior XData-tagged inserts are erased).
  7. M6 — Checklist for sheets/publish (extend when your Civil API path is fixed).
  8. M7 — BOQ CSV from volumes + marking layer solids + signage layer counts.

Concepts

Big ideas in a few lines

  1. Alignment — The horizontal path of the road centreline (or a reference line), like a bird’s-eye polyline with stationing along it.
  2. PI (point of intersection) — A point where straight segments meet; your CSV lists coordinates for each PI so the computer can build the horizontal layout.
  3. Profile — Heights along the alignment: “at station 100 m, the road is 212.5 m above datum,” and so on.
  4. PVI — A point on the profile where grade or curve data changes (the vertical cousin of a PI).
  5. Corridor — A 3D road model built by sweeping an assembly (lane + shoulder + slopes) along the alignment and profile.
  6. EG / FG — Existing ground surface vs finished grade (design) surface — used together for earthwork volumes.
  7. BOQ (bill of quantities) — A table of measured quantities tied to pay items for contracts; the scripts help roll up numbers, but humans still approve them.
Remember: The automation applies your numbers and names from CSV/JSON. It does not replace design checks, standards, or site knowledge.

def get_polyline_points(tr, obj_id):
    """Return list of Point3d from a 3dPolyline or FeatureLine."""
    obj = tr.GetObject(obj_id, OpenMode.ForRead)
    pts = []
    if isinstance(obj, FeatureLine):
        for i in range(obj.PointsCount):
            pts.append(obj.GetPoint(i))
    elif isinstance(obj, Polyline3d):
        for vtx_id in obj:
            vtx = tr.GetObject(vtx_id, OpenMode.ForRead)
            pts.append(vtx.Position)
    return pts


def segment_lengths(pts):
    """Return list of (cumulative_start, length, pt_a, pt_b)."""
    segs = []
    cum = 0.0
    for i in range(len(pts) - 1):
        a, b = pts[i], pts[i+1]
        d = a.DistanceTo(b)
        segs.append((cum, d, a, b))
        cum += d
    return segs, cum


def lerp_point(a, b, t):
    """Linearly interpolate between two Point3d values."""
    return Point3d(
        a.X + (b.X - a.X) * t,
        a.Y + (b.Y - a.Y) * t,
        a.Z + (b.Z - a.Z) * t,
    )


def point_at_distance(segs, dist):
    """Return Point3d at a given cumulative distance along segments."""
    for (cum_s, seg_d, a, b) in segs:
        if dist <= cum_s + seg_d + 1e-6:
            t = (dist - cum_s) / seg_d if seg_d > 0 else 0
            return lerp_point(a, b, max(0, min(1, t)))
    return segs[-1][3]   # end of line


def build_dash_intervals(total_len, dash, gap):
    """
    Return list of (start_dist, end_dist) for each ON segment.
    If dash == 0: single interval spanning full length (solid).
    """
    intervals = []
    if dash == 0:
        intervals.append((0.0, total_len))
        return intervals
    cycle = dash + gap
    pos = 0.0
    while pos < total_len:
        end = min(pos + dash, total_len)
        if end - pos > 0.01:   # skip slivers
            intervals.append((pos, end))
        pos += cycle
    return intervals


def create_box_solid(db, tr, msp, pt_a, pt_b, width, thickness):
    """
    Create a 3D Solid box representing one dash segment.
    The box is oriented along the vector pt_a → pt_b,
    width is perpendicular in the XY plane, thickness is in Z.
    Returns the ObjectId of the created solid.
    """
    vec = Vector3d(pt_b.X - pt_a.X, pt_b.Y - pt_a.Y, pt_b.Z - pt_a.Z)
    length = vec.Length
    if length < 0.001:
        return None
    vec = vec.GetNormal()

    # Local coordinate system
    up    = Vector3d.ZAxis
    right = vec.CrossProduct(up).GetNormal()   # width direction
    if right.Length < 0.5:                       # vec nearly vertical
        right = Vector3d.XAxis

    hw = width / 2.0

    # Eight corners of the box
    corners = [
        pt_a + right * hw,
        pt_a - right * hw,
        pt_b - right * hw,
        pt_b + right * hw,
    ]

    # Build solid via Acad3DSolid (box method is simpler via COM;
    # here we use the managed API's AddBox then transform)
    solid = Autodesk.AutoCAD.DatabaseServices.Solid3d()
    solid.CreateBox(length, width, thickness)

    # Build transform: origin at midpoint, X along vec
    mid = Point3d(
        (pt_a.X + pt_b.X) / 2,
        (pt_a.Y + pt_b.Y) / 2,
        (pt_a.Z + pt_b.Z) / 2 + thickness / 2,
    )
    mat = Matrix3d.AlignCoordinateSystem(
        Point3d.Origin, Vector3d.XAxis, Vector3d.YAxis, Vector3d.ZAxis,
        mid, vec, right, Vector3d.ZAxis
    )
    solid.TransformBy(mat)

    solid.Layer = MARK_LAYER
    solid.ColorIndex = 7   # White

    msp.AppendEntity(solid)
    tr.AddNewlyCreatedDBObject(solid, True)
    return solid.ObjectId


def delete_old_solid(db, tr, msp, src_handle_str):
    """
    Find and erase a previously generated solid that stores
    src_handle_str in its XData (app: ROAD_MARK_SRC).
    """
    for entity_id in msp:
        ent = tr.GetObject(entity_id, OpenMode.ForRead)
        xdata = ent.GetXDataForApplication(XDATA_APP)
        if xdata is None:
            continue
        for tv in xdata:
            if tv.TypeCode == 1000 and tv.Value == src_handle_str:
                ent.UpgradeOpen()
                ent.Erase()
                break

def run():
    doc = Application.DocumentManager.MdiActiveDocument
    db  = doc.Database
    ed  = doc.Editor

    if mark_type not in STANDARDS:
        return f"ERROR: Unknown marking type '{mark_type}'"

    dash, gap, default_w, thickness = STANDARDS[mark_type]
    width = float(width_override) if width_override else default_w

    with doc.LockDocument():
        with db.TransactionManager.StartTransaction() as tr:
            bt  = tr.GetObject(db.BlockTableId, OpenMode.ForRead)
            msp = tr.GetObject(bt[BlockTableRecord.ModelSpace], OpenMode.ForWrite)

            # Unwrap Dynamo element to get ObjectId
            try:
                obj_id = source_obj.InternalObjectId
            except:
                return "ERROR: Could not unwrap source object."

            src_handle = tr.GetObject(obj_id, OpenMode.ForRead).Handle.ToString()

            # Remove previous solid generated from this source line
            delete_old_solid(db, tr, msp, src_handle)

            # Ensure layer exists
            lt = tr.GetObject(db.LayerTableId, OpenMode.ForWrite)
            if not lt.Has(MARK_LAYER):
                from Autodesk.AutoCAD.DatabaseServices import LayerTableRecord
                ltr = LayerTableRecord()
                ltr.Name = MARK_LAYER
                lt.Add(ltr)
                tr.AddNewlyCreatedDBObject(ltr, True)

            # Ensure XData app is registered
            rat = tr.GetObject(db.RegAppTableId, OpenMode.ForWrite)
            if not rat.Has(XDATA_APP):
                from Autodesk.AutoCAD.DatabaseServices import RegAppTableRecord
                ra = RegAppTableRecord()
                ra.Name = XDATA_APP
                rat.Add(ra)
                tr.AddNewlyCreatedDBObject(ra, True)

            # Get points along the source line
            pts = get_polyline_points(tr, obj_id)
            if len(pts) < 2:
                return "ERROR: Source line has fewer than 2 vertices."

            segs, total_len = segment_lengths(pts)
            intervals = build_dash_intervals(total_len, dash, gap)

            if not intervals:
                return "ERROR: No dash intervals generated — line too short?"

            solid_ids = []
            for (s, e) in intervals:
                pa = point_at_distance(segs, s)
                pb = point_at_distance(segs, e)
                sid = create_box_solid(db, tr, msp, pa, pb, width, thickness)
                if sid:
                    solid_ids.append(sid)

            if not solid_ids:
                return "ERROR: No solids created."

            # Boolean-union all solids into the first one
            base = tr.GetObject(solid_ids[0], OpenMode.ForWrite)
            for i in range(1, len(solid_ids)):
                tool = tr.GetObject(solid_ids[i], OpenMode.ForWrite)
                base.BooleanOperation(
                    Autodesk.AutoCAD.DatabaseServices.BooleanOperationType.BoolUnite,
                    tool
                )
                tool.Erase()

            # Stamp XData on the unified solid so we can find it on re-run
            rb = ResultBuffer(
                TypedValue(1001, XDATA_APP),       # app name
                TypedValue(1000, src_handle),       # source handle string
                TypedValue(1000, mark_type),        # type for reference
            )
            base.XData = rb

            tr.Commit()

    return (
        f"OK: {mark_type} marking created. "
        f"{len(intervals)} dashes unified into 1 solid. "
        f"Total length: {total_len:.2f} m, Width: {width:.3f} m."
    )

OUT = run()
Deep Dive

Key techniques explained

Boolean Union — the "single solid" trick

Each dash is created as an independent Solid3d. After all dashes exist in the database, a loop calls BooleanOperation(BoolUnite, tool) on the first solid against each subsequent one, then erases the consumed tool. The result is a single Solid3d with internal history stored as BREP shells.

Dynamic link via XData

XData is AutoCAD's per-object key-value store. The script writes the source line's Handle (a permanent string ID, unlike ObjectIds which can change) into the solid's XData under a registered application name. On re-run, delete_old_solid() scans model space, finds the solid whose XData matches the current source handle, erases it, then regenerates fresh — giving the illusion of a live parametric link.

Coordinate transform for orientation

CreateBox() always produces a world-axis-aligned box. Matrix3d.AlignCoordinateSystem() rotates and translates it so its local X-axis aligns with the dash direction vector, placing it correctly in 3D space even on curved or graded alignments.

For curved alignments: Increase the sampling resolution in get_polyline_points() by subdividing each segment. Feature lines from a corridor already carry dense vertex arrays — this is why they're preferred over simple polylines for curved roads.

Improving for real production use

Curved segments
Sample each polyline segment at every 0.5 m rather than just at vertices. This approximates curves and avoids flat-chord artifacts on tight radii.
Layer + color by type
Assign different layers and True Colors per marking type to support automated plan production and quantity takeoffs in Civil 3D.
Double lines (M3)
Run the script twice with ±offset to the source line, or add an offset parameter that creates two parallel solids and unions them together.
Dynamo Player params
Expose mark_type and width via Dynamo Player's input panel so users never open the full editor — just click Run in the Player window.

# Inputs (Dynamo):
#   IN[0] : csv_path (string)

import clr, csv
clr.AddReference('AcMgd')
clr.AddReference('AcDbMgd')
clr.AddReference('AeccDbMgd')

from Autodesk.AutoCAD.ApplicationServices import Application
from Autodesk.AutoCAD.DatabaseServices import OpenMode, Transaction
from Autodesk.AutoCAD.Geometry         import Point2d, Point3d
from Autodesk.Civil.DatabaseServices  import (
    Alignment, AlignmentType, AlignmentSubEntityType
)

csv_path           = IN[0]
alignment_name     = IN[1]
criteria_set_name  = IN[2]
start_station      = float(IN[3] if IN[3] else 0.0)

def read_pis(path):
    pis = []
    with open(path, 'rb') as f:
        reader = csv.DictReader(f)
        for row in reader:
            pis.append({
                'id'     : row['pi_id'],
                'pt'     : Point2d(float(row['easting']), float(row['northing'])),
                'radius' : float(row['radius_m']),
                'spi'    : float(row['spiral_in_m']),
                'spo'    : float(row['spiral_out_m']),
                'speed'  : float(row['design_speed_kph']),
            })
    return pis


def validate(pis):
    # Simple QA: min radius vs design speed (placeholder rule)
    issues = []
    for p in pis:
        if p['radius'] > 0 and p['radius'] < p['speed'] * 2.5:
            issues.append("PI %s radius %.1f below 2.5*V threshold" % (p['id'], p['radius']))
    return issues


def build_alignment():
    doc = Application.DocumentManager.MdiActiveDocument
    db  = doc.Database
    civdoc = Autodesk.Civil.ApplicationServices.CivilApplication.ActiveDocument

    pis = read_pis(csv_path)
    issues = validate(pis)

    with doc.LockDocument():
        with db.TransactionManager.StartTransaction() as tr:
            # Resolve layer + styles (already configured in template DWG)
            site_id  = civdoc.GetSiteIds()[0] if civdoc.GetSiteIds().Count > 0 else Autodesk.AutoCAD.DatabaseServices.ObjectId.Null
            style_id = civdoc.Styles.AlignmentStyles["Standard"]
            label_id = civdoc.Styles.LabelSetStyles.AlignmentLabelSetStyles["Standard"]

            align_id = Alignment.Create(
                civdoc, alignment_name, site_id,
                "0", style_id, label_id,
                AlignmentType.Centerline
            )
            align = tr.GetObject(align_id, OpenMode.ForWrite)
            align.StationIndexingIncrement = 100.0
            align.ReferenceStation         = start_station
            align.DesignSpeedSummary.Clear()

            entities = align.Entities
            prev = None
            for i, pi in enumerate(pis):
                if i == 0:
                    prev = pi['pt']
                    continue
                if pi['radius'] == 0 or i == len(pis)-1:
                    entities.AddFixedLine(prev, pi['pt'])
                else:
                    nxt = pis[i+1]['pt'] if i+1 < len(pis) else pi['pt']
                    if pi['spi'] > 0 and pi['spo'] > 0:
                        entities.AddFreeSpiralCurveSpiral(
                            prev, pi['pt'], nxt,
                            pi['spi'], pi['radius'], pi['spo']
                        )
                    else:
                        entities.AddFreeCurve(prev, pi['pt'], nxt, pi['radius'])
                prev = pi['pt']

            # Stamp design speed across whole alignment
            align.DesignSpeedSummary.AddDesignSpeed(start_station, pis[0]['speed'])

            tr.Commit()

    msg = "OK: alignment '%s' created with %d PIs" % (alignment_name, len(pis))
    if issues:
        msg += " | WARNINGS: " + "; ".join(issues)
    return msg

OUT = build_alignment()

# M2 — Build finished-grade profile from PVI CSV
# Inputs (Dynamo):
#   IN[0] : csv_path

#   IN[2] : profile_name           e.g. "FG"

import clr, csv
clr.AddReference('AcDbMgd'); clr.AddReference('AcMgd'); clr.AddReference('AeccDbMgd')

from Autodesk.AutoCAD.ApplicationServices import Application
from Autodesk.AutoCAD.DatabaseServices import OpenMode
from Autodesk.Civil.DatabaseServices  import (
    Alignment, Profile, ProfileType, VerticalCurveType
)
from Autodesk.Civil.ApplicationServices import CivilApplication

csv_path        = IN[0]
align_name      = IN[1]
profile_name    = IN[2]
max_grade_pct   = float(IN[3])
min_grade_pct   = float(IN[4])


def read_pvis(path):
    rows = []
    with open(path, 'rb') as f:
        for r in csv.DictReader(f):
            rows.append({
                'sta'  : float(r['station_m']),
                'elev' : float(r['elevation_m']),
                'kc'   : float(r['k_crest']),
                'ks'   : float(r['k_sag']),
                'len'  : float(r['curve_length_m']),
            })
    rows.sort(key=lambda x: x['sta'])
    return rows


def grade_qa(pvis):
    issues = []
    for i in range(1, len(pvis)):
        d = pvis[i]['sta'] - pvis[i-1]['sta']
        if d <= 0: continue
        g = (pvis[i]['elev'] - pvis[i-1]['elev']) / d * 100.0
        if abs(g) > max_grade_pct:
            issues.append("Grade %.2f%% between %.1f-%.1f exceeds max %.1f%%"
                          % (g, pvis[i-1]['sta'], pvis[i]['sta'], max_grade_pct))
        if 0 < abs(g) < min_grade_pct:
            issues.append("Grade %.2f%% below drainage min %.1f%%" % (g, min_grade_pct))
    return issues


def build_profile():
    doc = Application.DocumentManager.MdiActiveDocument
    db  = doc.Database
    civdoc = CivilApplication.ActiveDocument

    pvis   = read_pvis(csv_path)
    issues = grade_qa(pvis)

    align_id = None
    for aid in civdoc.GetAlignmentIds():
        align_id = aid
        if Alignment.GetAlignmentName(aid) == align_name: break
    if align_id is None:
        return "ERROR: alignment '%s' not found" % align_name

    with doc.LockDocument():
        with db.TransactionManager.StartTransaction() as tr:
            align = tr.GetObject(align_id, OpenMode.ForWrite)
            style_id = civdoc.Styles.ProfileStyles["Design Profile"]
            label_id = civdoc.Styles.LabelSetStyles.ProfileLabelSetStyles["Standard"]

            prof_id = Profile.CreateByLayout(
                profile_name, align_id, "0", style_id, label_id
            )
            prof = tr.GetObject(prof_id, OpenMode.ForWrite)
            ents = prof.Entities

            # Add tangents
            for i in range(1, len(pvis)):
                a, b = pvis[i-1], pvis[i]
                ents.AddFixedTangent(a['sta'], a['elev'], b['sta'], b['elev'])

            # Add vertical curves at internal PVIs
            for i in range(1, len(pvis)-1):
                p = pvis[i]
                if p['len'] <= 0: continue
                ents.AddFreeVerticalCurveByPVILength(
                    p['sta'], p['len'], VerticalCurveType.Parabolic
                )

            tr.Commit()

    msg = "OK: profile '%s' built with %d PVIs" % (profile_name, len(pvis))
    if issues:
        msg += " | QA: " + "; ".join(issues)
    return msg

                    elif "_R" in tgt.SubassemblyName and t_right:

def station_to_point(align, sta, off, side):
    sign = -1.0 if side.upper() == "L" else 1.0
    pt = align.PointLocation(sta, sign * off)
    ang = align.GetTangentAngle(sta)
    return Point3d(pt.X, pt.Y, 0), ang


def place_signs():
    doc = Application.DocumentManager.MdiActiveDocument
    db  = doc.Database
    civdoc = CivilApplication.ActiveDocument

    rows = list(csv.DictReader(open(csv_path, 'rb')))

    align_id = None
    for aid in civdoc.GetAlignmentIds():
        if Alignment.GetAlignmentName(aid) == align_name:
            align_id = aid; break
    if align_id is None: return "ERROR: alignment not found"

    placed, missing = 0, []

    with doc.LockDocument():
        with db.TransactionManager.StartTransaction() as tr:
            bt  = tr.GetObject(db.BlockTableId, OpenMode.ForRead)
            msp = tr.GetObject(bt[BlockTableRecord.ModelSpace], OpenMode.ForWrite)
            align = tr.GetObject(align_id, OpenMode.ForRead)

            for r in rows:
                blk_name = r['block_name']
                if not bt.Has(blk_name):
                    missing.append(blk_name); continue

                pt, ang = station_to_point(
                    align,
                    float(r['station_m']),
                    float(r['offset_m']),
                    r['side']
                )
                rot = math.radians(float(r['rotation_deg'])) if r['rotation_deg'] else ang
                br  = BlockReference(pt, bt[blk_name])
                br.Rotation = rot
                br.Layer    = layer_name
                msp.AppendEntity(br)
                tr.AddNewlyCreatedDBObject(br, True)
                placed += 1

            tr.Commit()

    msg = "OK: %d signs placed" % placed
    if missing:
        msg += " | missing blocks: " + ", ".join(sorted(set(missing)))
    return msg

def corridor_quantities(tr, civdoc):
    # Pull material quantities by name from corridor QTO
    out = {}

        cor = tr.GetObject(cid, OpenMode.ForRead)

def marking_quantities(tr, db):
    # Sum lengths by XData type stamp set in the marking module
    out = {}
    msp = tr.GetObject(
        tr.GetObject(db.BlockTableId, OpenMode.ForRead)[BlockTableRecord.ModelSpace],
        OpenMode.ForRead)
    for oid in msp:
        ent = tr.GetObject(oid, OpenMode.ForRead)
        if ent.Layer != marking_layer: continue
        xd = ent.GetXDataForApplication("ROAD_MARK_SRC")
        if xd is None: continue
        # Find mark_type tag (third TypedValue per writer in original module)
        mtype = None
        for tv in xd:
            if tv.TypeCode == 1000 and tv.Value in ("M1","M2","M3","M4","M5","M6","M7","M8","M9","M10"):
                mtype = tv.Value
        if mtype is None: continue
        # Length approximated from bounding box X-extent (or true length via solid edges)
        bb  = ent.GeometricExtents
        ln  = (bb.MaxPoint.X - bb.MinPoint.X)
        out[mtype] = out.get(mtype, 0.0) + ln
    return out

Governance, QA & safe automation

Automation only stays useful if it stays trustworthy. These rules keep the pipeline auditable and reversible.

G1
Inputs are versioned
All CSVs and project.yaml live in git. Every drawing revision is reproducible from the input set at that commit.
G2
QA outputs are persisted
M1 grade flags, M2 K-value warnings, and M4 volume reports go to ./out/qa/ on every run for review.
G3
Idempotent re-runs
Every module deletes its previous output (via name or XData) before regenerating. No silent duplication, no orphan geometry.
G4
Human gates stay in place
Designer sign-off on QA report, checker review of BOQ, and PM approval of the published sheet set. Automation accelerates — it doesn't replace approval.
Failure mode to expect: CSV typos. Add a column validator at the top of each module (already done in M1/M2) and refuse to run if required columns are missing or non-numeric. Loud failure beats silent wrong geometry.
