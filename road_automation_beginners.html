Concepts

Big ideas in a few lines

  1. Alignment — The horizontal path of the road centreline (or a reference line), like a bird’s-eye polyline with stationing along it.
  2. PI (point of intersection) — A point where straight segments meet; your CSV lists coordinates for each PI so the computer can build the horizontal layout.
  3. Profile — Heights along the alignment: “at station 100 m, the road is 212.5 m above datum,” and so on.
  4. PVI — A point on the profile where grade or curve data changes (the vertical cousin of a PI).
  5. Corridor — A 3D road model built by sweeping an assembly (lane + shoulder + slopes) along the alignment and profile.
  6. EG / FG — Existing ground surface vs finished grade (design) surface — used together for earthwork volumes.
  7. BOQ (bill of quantities) — A table of measured quantities tied to pay items for contracts; the scripts help roll up numbers, but humans still approve them.
Remember: The automation applies your numbers and names from CSV/JSON. It does not replace design checks, standards, or site knowledge.

Who does what

Two common roles

I only fill tables and press Run

  • Use header-only files in civil3d_automation/csv/templates/ as a starting copy into csv/.
  • Edit config/project.json (start from project.example.json if you like). Field meanings: project.schema_notes.md.
  • Run civil3d_automation/tools/run_preflight.bat before Civil 3D to catch missing columns or wrong paths.
  • In Civil 3D, use Dynamo Player with the absolute path to project.json — see the user guide → run order.

I set up the drawing template once

  • Layers, Civil styles, label sets, assemblies, blocks for signs, and surface names must match what project.json and the CSVs reference.
  • Checklist: template/README.md.
Terms

Glossary (quick lookup)

TermIn plain language
Station / chainageDistance along the alignment from a defined start; used to place profile points and signs.
PIHorizontal geometry point — often a corner where direction changes.
PVIVertical geometry point on the profile (height and curve data).
AssemblyCross-section “recipe” of lanes, shoulders, and slopes used to build the corridor.
Corridor regionA length of corridor where a given assembly applies between two stations.
EG / FGExisting ground vs finished design surface — compare for cut/fill volumes.
Pay itemA contract line item code; payitems.csv maps model outputs to those codes.
XDataHidden tags on objects so scripts can find or replace their own geometry safely on re-run.
Dynamo PlayerCivil 3D UI to run saved Dynamo graphs with simple inputs — your “popup” for paths and options.
Support

When the script says ERROR:

The Dynamo Watch node shows messages starting with OK: or ERROR:. Below is a plain-language map for common cases.

You might seeWhat it usually meansWhat to try
ERROR: project.json not foundM8 path is wrong or the file moved.Use the preflight helper’s Copy project.json path, or browse again in Dynamo.
ERROR: CSV not foundA path in project.json does not resolve to a real file.Open csv/ in Explorer; fix paths.* in JSON; run preflight.
ERROR: Need at least 2 PI rowsAlignment table is empty or only one point.Add at least two coordinate rows to alignment_pi.csv.
ERROR: Alignment not found / Corridor not foundName in JSON does not match the drawing.Check names.alignment / names.corridor against Prospector, or run M1/M3 first.
ERROR: Profile … already existsRe-run blocked so you do not stack duplicate profiles.Rename or delete the profile in Civil 3D, then run again.
ERROR: Surfaces not found (EG=… FG=…)Surface names in JSON do not match TINs in the drawing.Rename surfaces or update names.surface_eg / surface_fg.
ERROR: TinVolumeSurface.Create failedVolume surface could not be built (surface type, version, or data issue).Confirm both surfaces are valid for volumes in your Civil 3D version; see API_NOTES.md.
ERROR: BaselineRegions.Add failedRegion name clash, bad station range, or assembly name wrong.Check section_widths.csv and that names.assembly exists in the template.
ERROR: payitems CSV path missing or not foundM7 cannot load the pay-item map.Fix paths.payitems and file presence.
ERROR: output BOQ CSV path requiredBOQ output path was empty in the script inputs.Set paths.boq_csv in project.json for M8.
If preflight reports missing CSV columns, compare your file’s header row to the header-only templates in csv/templates/ — one renamed column is enough to break a module.
